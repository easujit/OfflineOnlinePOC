Prompt:

You are a technical architect in software development.

Context:
 In hill stations, hospitals face serious problems when the internet goes down because of heavy rain, landslides, earthquakes, or shutdowns. The hospital needs a web application that must work 24×7 even without the internet. Doctors, nurses, and staff should continue their work without any problem.

Challenges:

  1. The Internet may be down for 2–3 days continuously.
  2. The hospital system generates a huge amount of data (patient records, lab reports, images, prescriptions).
  3. Creating separate data stores for each hospital is too costly.
  4. Patients’ lives are at risk if the system stops working when the internet is not available.

Your task:

  • Design and explain a complete architecture for such a system.
  • Show how the web application can keep working offline for many days.
  • Explain how to handle large amounts of data locally and sync it later when the internet is back.
  • Give a sync and conflict resolution strategy so that no data is lost or duplicated.
  • Suggest cost-effective approaches so that it is scalable and hospitals don’t need expensive infrastructure.
  • Use simple language so that even a non-expert can understand.
  • Provide examples, diagrams (if possible), and step-by-step explanation.
